\subsection{Project Summary}

This project simulates a 12-month software development timeline for the LA City Sidewalk Assessment (Ascent) system. The system supports a rover-based sidewalk inspection workflow that collects LiDAR and point-cloud data, segments sidewalk joints and cracks using a U-Net deep learning model, measures vertical and horizontal displacement, and produces review-ready outputs for the LA City Bureau of Engineering (BOE) for ADA compliance assessment.

The simulation is divided into four snapshots of approximately three simulated months each, with each snapshot adding a major capability: foundation, displacement pipeline, semi-autonomous navigation, and final delivery.

\section{Snapshot 1 -- Project Foundation}
%==========================================================

\textit{Simulated Months 1--3}

\subsection{Objective}

Establish the repository, Jira board, baseline architecture, and initial documentation for the LA City Sidewalk Assessment simulation. By the end of Snapshot 1 the project must be organizationally complete even though no major feature has been implemented.

\subsection{Planned Work Items}

\begin{itemize}[leftmargin=*]
  \item Reorganize and document the repository structure
  \item Create Jira epics and Snapshot 1 sprint
  \item Draft SRS, SDD, README/User Manual, Design Spec, and workflow diagram
  \item Define team responsibilities and initial risk register
  \item Create Docker compose draft
  \item Create initial manual testing template
\end{itemize}

\subsection{Definition of Done}

\begin{itemize}[leftmargin=*]
  \item All required \texttt{.tex} files exist
  \item Workflow diagram exists
  \item Jira board and sprint exist
  \item Docker compose file exists
  \item Each team member has at least one merged contribution
\end{itemize}

\subsection{Snapshot 1 Reflection}

Completed: Repository structure, Jira setup, SRS/SDD drafts, workflow diagram, Docker planning, and role assignments were completed during Snapshot 1.

Deferred: Advanced navigation architecture and detailed displacement validation were deferred to later snapshots.

Lessons learned: Early repository organization and document standardization significantly reduced merge conflicts and formatting inconsistencies.
