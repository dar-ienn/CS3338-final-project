\section{Project Objective}
This project simulates the software development timeline for a sidewalk assessment rover system that helps identify unsafe or ADA non-compliant sidewalk segments.

\section{Why This Software Matters}
The system supports safer sidewalks by organizing data collection, image processing, displacement measurement, testing, and review workflows.

\section{How to Download}
\begin{verbatim}
git clone https://github.com/dar-ienn/CS3338-final-project.git
\end{verbatim}

\section{How to Run With Docker}
\begin{verbatim}
cd docker
docker compose config
docker compose build
docker compose up
\end{verbatim}

\section{How to View Outputs}
Processed sample outputs are stored in tests/sample_outputs and storage/processed.

\section{Known Limitations}

\begin{itemize}
    \item The project is a simulated software engineering workflow and does not represent a fully deployed rover system.
    \item U-Net segmentation and navigation systems are documented architecturally rather than fully implemented.
    \item Docker services represent planned deployment architecture.
    \item Real-world rover hardware integration is outside the scope of this CS3338 simulation project.
\end{itemize}
